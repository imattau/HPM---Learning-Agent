# SP21: Toddler Sentence Generator — Design Specification

## 1. Overview and Rationale

The **Toddler Sentence Generator** (SP21) tests the HPM AI's capacity for "Top-Down Synthesis." Moving away from bottom-up observation, this experiment evaluates if the system can use its distributed, multi-domain world models to *generate* structurally and semantically correct simple sentences.

To achieve this, we implement a highly discrete **"Structural Octet"** of HFN domains. A sentence is generated by finding an alignment (a multi-forest path) across these independent knowledge bases.

## 2. The 8-Domain "Structural Octet"

The system is seeded with 8 distinct `TieredForest` instances, each representing a specific cognitive or linguistic capability.

### 2.1 The Lexical Layer
1.  **Token Forest**: Raw word strings (e.g., "mum", "eat", "apple").
2.  **Morphological Forest**: Tense and plurality modifiers (e.g., `[+PAST]`, `[+PLURAL]`).

### 2.2 The Semantic Layer
3.  **Identity Forest**: Persistent entities (e.g., `Entity_Mum`, `Entity_Apple_1`).
4.  **Categorical Forest**: Hierarchical taxonomy (e.g., `[Person]`, `[Food]`).
5.  **Affordance Forest**: Action capabilities (e.g., `[Person]` can `[Eat]`; `[Food]` can be `[Eaten]`).

### 2.3 The Relational Layer
6.  **Syntactic Forest**: Grammatical slots (e.g., `[Subject]`, `[Verb]`, `[Object]`).
7.  **Relational Forest**: Cross-domain bindings (e.g., `Entity_Mum` $\leftrightarrow$ `[Person]`).
8.  **Narrative Forest**: Event templates (e.g., `[Agent] -> [Action] -> [Patient]`).

## 3. The Generative Architecture (Top-Down Synthesis)

We will build a `SentenceSynthesizer` that acts as an "Inverted Observer."

**The Goal**: Given a high-level "Mental Event" (e.g., `Event_Eat(Agent=Entity_Mum, Patient=Entity_Apple_1)`), generate the correct string array: `["Mum", "ate", "the", "apple"]`.

**The Algorithm (Constraint Satisfaction)**:
1.  **Retrieve Template**: Query the Narrative Forest for the `[Agent] -> [Action] -> [Patient]` template.
2.  **Validate Affordances**: Ensure `Entity_Mum` matches the `[Eat]` affordance. If invalid (e.g., "Apple ate Mum"), abort.
3.  **Assign Syntax**: Map the Agent to the `[Subject]` slot, Action to `[Verb]`, Patient to `[Object]`.
4.  **Apply Morphology**: 
    *   If the event is marked `Past`, the Morphological Forest flags the `[Verb]` slot for `[+PAST]`.
5.  **Resolve Tokens**:
    *   Query the Relational Forest to find the Token IDs bound to the Entities and Actions.
    *   `Entity_Mum` $\rightarrow$ "mum"
    *   `Action_Eat` + `[+PAST]` $\rightarrow$ "ate"
    *   `Entity_Apple` $\rightarrow$ "apple" (Syntax Forest injects "the" or "an" based on Category constraints).

## 4. The Experiment Setup

### 4.1 Data Structures
We will create a small, hardcoded set of HFN priors for the 8 forests. 
*   **Entities**: Mum, Dog, Apple, Ball.
*   **Actions**: Eat, Chase.
*   **Categories**: Person, Animal, Food, Toy.

### 4.2 The "Toddler Prompts"
We will feed the synthesizer a list of "Mental Goals":
1.  `Goal_1`: `Event(Mum, Eat, Apple, PAST)` $\rightarrow$ Expect: "Mum ate the apple"
2.  `Goal_2`: `Event(Dog, Chase, Ball, PRESENT)` $\rightarrow$ Expect: "The dog chases the ball"
3.  `Goal_3`: `Event(Apple, Eat, Mum, PAST)` $\rightarrow$ Expect: **Exception** (Affordance Violation)

## 5. Smoke Testing (Structural Integrity)

Before running the full synthesis prompts, the system must pass these three smoke tests:

1.  **Direct Path Retrieval**: Verify that `Entity_Mum` can resolve to the string "mum" by traversing the `Relational -> Token` path.
2.  **Affordance Guard**: Verify that `Affordance_Eat` correctly validates `Entity_Mum` as a valid Agent and `Entity_Apple` as a valid Patient, but rejects the inverse.
3.  **Template Slotting**: Verify that the `Narrative_Event` HFN correctly identifies its 3 empty child slots: `[Agent]`, `[Action]`, and `[Patient]`.

## 6. Evaluation Metrics

1.  **Grammatical Accuracy**: Are the correct syntactic slots and morphological modifiers applied?
2.  **Semantic Validity**: Does the Affordance Forest correctly block nonsensical events (e.g., "Apple ate mum")?
3.  **Cross-Forest Traversal**: Does the synthesizer successfully navigate the DAG edges from the Narrative Forest all the way down to the Token Forest?
4.  **Smoke Pass Rate**: Percentage of structural unit tests passed before main execution.

## 7. Implementation Roadmap

1.  **Forest Builder**: Create `build_octet_priors()` to initialize the 8 `TieredForest` instances.
2.  **Smoke Test Suite**: Implement the three integrity checks in `experiment_toddler_generator.py`.
3.  **Synthesizer Engine**: Implement `SentenceSynthesizer.generate(goal)` to perform top-down resolution.
4.  **Experiment Script**: Run the full "Toddler Prompts" and validate outputs.
