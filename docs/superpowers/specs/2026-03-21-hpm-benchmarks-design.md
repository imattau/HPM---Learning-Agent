# HPM Benchmarks Design Specification

**Date:** 2026-03-21
**Status:** Draft v1

---

## Overview

Three standalone benchmark scripts that evaluate the HPM agent against industry-standard AI evaluation paradigms. Each script prints a results table to stdout and is independently runnable. All three share a common utilities module.

---

## File Structure

```
benchmarks/
  common.py                 # shared agent factory, config defaults, table printer
  elegance_recovery.py      # Benchmark 1: Symbolic Regression
  structural_immunity.py    # Benchmark 2: OOD Robustness / Concept Drift
  substrate_efficiency.py   # Benchmark 3: MDL / Pareto Efficiency
```

**Target runtime:** Under 2 minutes per script on a laptop.

**Benchmark parameters (shared defaults):**
- `feature_dim = 16`
- `beta_comp = 0.1` (enables `compress_mean` output)
- `T_recomb = 50`, `recomb_cooldown = 25`
- Evaluation frequency: every 20 steps (for heavy metrics)

---

## 1. `common.py`

Shared utilities imported by all three benchmark scripts.

### Contents

**`BENCH_CONFIG`** — base `AgentConfig` kwargs dict tuned for benchmarks:
```python
BENCH_CONFIG = dict(
    feature_dim=16,
    beta_comp=0.1,
    T_recomb=50,
    recomb_cooldown=25,
    kappa_0=0.5,
)
```

**`make_agent(feature_dim=16, **overrides) -> Agent`** — constructs an `Agent` with benchmark defaults. Accepts keyword overrides for per-benchmark customisation.

**`print_results_table(title, cols, rows)`** — prints a titled ASCII table to stdout. `cols` is a list of column header strings; `rows` is a list of dicts keyed by column name.

### Usage

```python
from benchmarks.common import make_agent, print_results_table
```

Each benchmark script is independently runnable: `python benchmarks/elegance_recovery.py`.

## 2. Benchmark 1: Elegance Recovery (`elegance_recovery.py`)

### Purpose

Tests whether the Recombination Operator produces compact patterns that structurally match a hidden mathematical law, given a data stream generated by that law. Evaluates Level 5 pattern formation and recombination quality.

### Hidden Function

```
y = x² / (1 + x)
```

Sampled with `x ~ Uniform[-2, 2]`. At each step, the observation vector is:
```python
hash_vectorise(f"{x:.4f} {y:.4f}", feature_dim)
```

### Atomic Seeding

Before training, pre-seed the agent's store with 5 patterns whose `mu = hash_vectorise(expr, feature_dim)` for each of: `"x"`, `"1"`, `"x**2"`, `"x**3"`, `"1+x"`. All seeded with equal weight (`1/5`). This gives the Recombination Operator meaningful algebraic building blocks.

Seeded patterns are stored via `agent.store.save(pattern, weight, agent.agent_id)`.

### Run

- **Steps:** 1500
- **feature_dim:** 16

### Metric: Structural Recovery Rate (SRR)

After training, query the agent's store for the top-weighted pattern. Compute cosine similarity between that pattern's `mu` and `hash_vectorise("x**2 / (1 + x)", feature_dim)`.

```
SRR = cosine_similarity(top_pattern.mu, hash_vectorise("x**2 / (1 + x)", 16))
```

SRR ∈ [0, 1]. Values > 0.7 indicate structural convergence to the ground-truth law.

### Output

```
Elegance Recovery
─────────────────────────────────────────────────
Steps   Top Weight   Recomb Count   SRR    Result
1500    0.73         12             0.81   ✓ RECOVERED
```

`Result` = `✓ RECOVERED` if SRR > 0.7, else `✗ NOT RECOVERED`.
`Recomb Count` = total accepted recombinations during the run.

## 3. Benchmark 2: Structural Immunity (`structural_immunity.py`)

### Purpose

Tests whether κ_D (pattern density stickiness) protects consolidated patterns during a noise storm and enables clean recovery. Evaluates the Pattern Density mechanism under concept drift.

### 3-Phase Data Stream (1020 steps total)

| Phase | Steps | Signal |
|-------|-------|--------|
| 1 — Stable | 500 | Gaussian, fixed mean μ₀ (random unit vector), std=0.1 |
| 2 — Noise | 20 | Uniform random in [-1, 1]^d |
| 3 — Recovery | 500 | Same Gaussian signal as Phase 1 |

### Baseline Measurement

At end of Phase 1, record:
- `B_acc` — mean accuracy (mean of `mean_accuracy` over last 50 steps)
- `B_weight` — dominant pattern weight (top pattern weight at Phase 1 end)

### Recovery Criterion

T_rec = first step into Phase 3 where **both** hold simultaneously:
1. `mean_accuracy >= 0.9 * B_acc` (within 10% of pre-noise baseline)
2. `top_pattern_weight >= 0.5`

If neither criterion is met by end of Phase 3: `T_rec = ∞` (reported as `>500`).

### Output

```
Structural Immunity
──────────────────────────────────────────────────────────
Phase 1 Acc   Phase 2 Acc   Phase 3 Acc   T_rec   Result
0.84          0.21          0.79          47      ✓ IMMUNE
```

`Phase N Acc` = mean accuracy over that phase's steps.
`Result` = `✓ IMMUNE` if T_rec ≤ 100, `~ SLOW` if 100 < T_rec ≤ 500, `✗ FAILED` if T_rec = ∞.

## 4. Benchmark 3: Substrate Efficiency (`substrate_efficiency.py`)

### Purpose

Tests whether HPM discovers a compact representation of a redundant data stream, and compares its compression/accuracy trade-off against a GMM baseline on a Pareto frontier.

### Data Stream

3 overlapping Gaussian clusters in 16-dimensional space, sampled uniformly across clusters at each step. The optimal model needs only 3 components — data is structurally redundant.

Cluster means: 3 fixed random unit vectors scaled to norm 1.0. Cluster std: 0.3 each.

### HPM Run

- **Steps:** 2000
- **Eval frequency:** Every 20 steps, record `(step, mean_accuracy, compress_mean, n_patterns)`

### GMM Comparison

Fit `sklearn.mixture.GaussianMixture` at k = 1, 2, 3, 4, 5 on 500 samples drawn from the same 3-cluster distribution. For each k record:
- `BIC` — Bayesian Information Criterion (lower = better fit per parameter)
- `mean_log_likelihood` — mean log-likelihood on the 500 samples

### Pareto Metrics

**HPM (final snapshot, last eval step):**
- Complexity proxy: `1 - compress_mean` (lower compress_mean = more spread = more complex)
- Accuracy: `mean_accuracy`

**GMM (each k):**
- Complexity proxy: BIC normalised to [0, 1] across k values
- Accuracy: `mean_log_likelihood` normalised to [0, 1]

**Pareto frontier:** A model is on the frontier if no other model achieves both lower complexity and higher accuracy simultaneously.

### Output

```
Substrate Efficiency
──────────────────────────────────────────────────────────────────
        Model         Complexity   Accuracy   On Pareto Frontier?
HPM (final)           0.31         0.78       ✓
GMM k=1               0.12         0.41       ✓
GMM k=3               0.44         0.81       ✓
GMM k=5               0.62         0.82       ✗
```

- `hpm.patterns.gaussian.GaussianPattern` — atomic seeding in Benchmark 1

---

## 6. Testing Strategy

Benchmarks are integration scripts, not unit-tested. Correctness is validated by:
- **Smoke test:** each script runs to completion without error
- **Sanity checks (inline assertions):**
  - SRR ∈ [0, 1]
  - T_rec is a positive integer or `>500`
  - Pareto frontier contains at least 1 entry
- **No pytest tests** for benchmarks — they are end-to-end evaluation tools, not unit components

---

## 7. What Is NOT in Scope

- Multiple hidden functions for Elegance Recovery (single function is sufficient for prototype)
- Plotting / visualisation (console table output only)
- Saving results to disk / CSV
- Hyperparameter sweep across AgentConfig settings
- Parallelisation across benchmarks
- CI integration (benchmarks are too slow for automated test runs)
